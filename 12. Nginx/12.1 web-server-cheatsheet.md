# Web Servers — Beginner Friendly Cheatsheet

> A **web server** receives HTTP/HTTPS requests from clients (browsers) and serves back responses (HTML, JSON, files, etc.).  
> Web servers act as the **front door** of your application.

## Nginx

**What it is:** Lightweight, high-performance web server and reverse proxy.  
**Used for:** Serving static files, reverse proxying to app servers, load balancing.

### Key Concepts
| Term | Meaning |
|------|---------|
| Reverse Proxy | Sits in front of your app, forwards requests to it |
| Upstream | The backend server(s) Nginx forwards traffic to |
| Worker Process | Nginx handles many requests using async worker processes |
| `server {}` block | Defines a virtual host (one domain/site) |
| `location {}` block | Rules for how to handle specific URL paths |

### Essential Config Example
```nginx
# /etc/nginx/nginx.conf or /etc/nginx/sites-available/myapp

server {
    listen 80;                        # listen on port 80 (HTTP)
    server_name example.com;          # your domain

    location / {
        proxy_pass http://localhost:3000;   # forward to your app on port 3000
        proxy_set_header Host $host;
    }

    location /static/ {
        root /var/www/html;           # serve static files directly
    }
}
```

### Common Commands
```bash
nginx -t                    # test config for syntax errors
nginx -s reload             # reload config without downtime
systemctl start nginx       # start nginx
systemctl enable nginx      # start nginx on boot
systemctl status nginx      # check if running
tail -f /var/log/nginx/access.log    # watch live traffic
tail -f /var/log/nginx/error.log     # watch errors
```

### Use Cases
- **Reverse proxy** in front of Node.js / Python / Go apps
- **SSL termination** — handle HTTPS, pass plain HTTP to backend
- **Load balancer** — split traffic across multiple app instances
- **Static file server** — serve React/Vue builds directly

## Apache (httpd)

**What it is:** The original and most widely used web server. Very flexible.  
**Used for:** Hosting websites, PHP apps (WordPress), `.htaccess` rules.

### Key Concepts
| Term | Meaning |
|------|---------|
| VirtualHost | One Apache instance serving multiple websites |
| `.htaccess` | Per-directory config file (override main config) |
| Module | Plugin to add features (mod_rewrite, mod_ssl, mod_proxy) |
| DocumentRoot | Folder where website files are stored |
| `AllowOverride` | Whether `.htaccess` overrides are allowed |

### Essential Config Example
```apache
# /etc/apache2/sites-available/myapp.conf

<VirtualHost *:80>
    ServerName example.com
    DocumentRoot /var/www/html/myapp

    <Directory /var/www/html/myapp>
        AllowOverride All         # allow .htaccess rules
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
```

### Common Commands
```bash
apachectl configtest            # test config
apachectl graceful              # reload without dropping connections
systemctl start apache2         # start (Ubuntu/Debian)
systemctl start httpd           # start (CentOS/RHEL)
a2ensite myapp.conf             # enable a site config (Debian)
a2dissite myapp.conf            # disable a site config
a2enmod rewrite                 # enable a module
tail -f /var/log/apache2/error.log
```

### Use Cases
- Hosting **PHP applications** (WordPress, Laravel)
- **URL rewriting** with mod_rewrite
- Legacy enterprise apps that depend on `.htaccess`

## Caddy

**What it is:** Modern web server with **automatic HTTPS** out of the box.  
**Used for:** Simple deployments where you want SSL without manual cert management.

### Key Concepts
| Term | Meaning |
|------|---------|
| Automatic HTTPS | Caddy auto-gets Let's Encrypt certs for your domain |
| Caddyfile | Simple human-readable config file |
| Reverse Proxy | Built-in, easy to configure |
| Let's Encrypt | Free SSL certificate authority Caddy uses by default |

### Essential Config Example (Caddyfile)
```caddyfile
# /etc/caddy/Caddyfile

example.com {
    reverse_proxy localhost:3000    # proxy to your app
}

# Serve static files — HTTPS is automatic!
files.example.com {
    root * /var/www/files
    file_server
}
```

### Common Commands
```bash
caddy validate --config /etc/caddy/Caddyfile    # check config
caddy reload --config /etc/caddy/Caddyfile       # reload
systemctl start caddy
systemctl status caddy
journalctl -u caddy --follow                     # view logs
```

### Use Cases
- **Zero-config HTTPS** — great for small teams / quick deployments
- Replacing Nginx when simplicity matters more than fine-tuning
- **API gateways** in Docker/Kubernetes environments

## Quick Comparison

| Feature | Nginx | Apache | Caddy |
|---------|-------|--------|-------|
| Language/Ecosystem | Any | PHP, general | Any |
| Auto HTTPS | No (manual) | No (manual) | **Yes** |
| Config Style | Simple blocks | Verbose XML-like | Very simple |
| Performance | Very High | Medium | High |
| OS | Linux/Unix | Linux/Windows | Linux/Unix |
| Use as Reverse Proxy | Yes | Yes | Yes |

## Common Concepts

### Reverse Proxy
```
Client → [Nginx/Apache/Caddy] → [Your App on localhost:3000]
```
- Hides your app from direct internet access
- Handles SSL/HTTPS at the proxy level
- Adds security headers, rate limiting, caching

### Load Balancing (Nginx Example)
```nginx
upstream myapp {
    server app1:3000;
    server app2:3000;
    server app3:3000;
}

server {
    listen 80;
    location / {
        proxy_pass http://myapp;
    }
}
```

### SSL Termination
```
Client ──HTTPS──► Nginx (terminate SSL here) ──HTTP──► App Server
```
- Let Nginx handle the SSL overhead
- Your app sees plain HTTP internally
- Easier to manage one cert in one place

### Log Files to Know
| Server | Access Log | Error Log |
|--------|-----------|-----------|
| Nginx | `/var/log/nginx/access.log` | `/var/log/nginx/error.log` |
| Apache | `/var/log/apache2/access.log` | `/var/log/apache2/error.log` |
| Caddy | `journalctl -u caddy` | `journalctl -u caddy -p err` |

### HTTP Status Codes (Quick Ref)
| Code | Meaning | Common Cause |
|------|---------|--------------|
| 200 | OK | Success |
| 301/302 | Redirect | URL changed |
| 400 | Bad Request | Malformed request |
| 401 | Unauthorized | Login required |
| 403 | Forbidden | No permission |
| 404 | Not Found | Wrong URL / missing file |
| 500 | Internal Server Error | App crashed |
| 502 | Bad Gateway | Backend app is down |
| 503 | Service Unavailable | Server overloaded |

---

*Tip: Nginx is the most commonly used web server for containerized/cloud deployments. Caddy is great for small projects needing zero-config HTTPS. Apache is common in shared hosting and PHP environments.*
