# Bash & Shell Scripting — Cheatsheet

---

## Introduction

When you use a Linux terminal and type commands like `ls`, `cd`, or `echo` — you are using a **shell**.

A **shell** is a program that takes your commands and runs them on the operating system.

**Bash** (Bourne Again SHell) is the most popular shell on Linux and macOS. It is the default shell in most Linux systems.

**Shell scripting** means writing those commands in a file (called a script) so they run automatically — one after another — without you typing them manually every time.

Think of a shell script like a recipe:
- You write the steps once
- You run the file whenever you need it
- The computer follows the steps exactly

**Example of what a shell script can do:**
```
Every night at midnight:
  → Check if a backup folder exists
  → If not, create it
  → Copy all important files into it
  → Print "Backup done"
```

You write this once as a `.sh` file and it runs on its own.

### Step 1: Understand the Shell

When you open a terminal, you are inside a shell. The shell reads what you type and runs it.

```bash
echo "Hello"       # shell reads this → runs echo → prints Hello
ls                 # shell reads this → runs ls → lists files
```

**Bash** is one type of shell. Others include `sh`, `zsh`, `fish`. Bash is the most common on Linux.

To check which shell you are using:
```bash
echo $SHELL        # prints: /bin/bash
```

---

### Step 2: Write Your First Script

A script is just a plain text file with commands inside it.

**Create a file called `hello.sh`:**
```bash
#!/bin/bash

echo "Hello, World!"
```

**What each line means:**

| Line | Meaning |
|------|---------|
| `#!/bin/bash` | Called a **shebang** — tells the system "use Bash to run this file" |
| `echo "Hello, World!"` | Prints text to the terminal |

**Make it executable and run it:**
```bash
chmod +x hello.sh     # give the file permission to run
./hello.sh            # run it
```

Output:
```
Hello, World!
```

### Step 3: Use Variables

A **variable** is a name that holds a value. You use it to store things and reuse them.

```bash
#!/bin/bash

name="Alice"
age=25

echo "My name is $name"
echo "I am $age years old"
```

Output:
```
My name is Alice
I am 25 years old
```

**Rules for variables:**
- No spaces around `=` (wrong: `name = "Alice"`, right: `name="Alice"`)
- Use `$name` to read the value (with the `$` sign)
- Use quotes around strings to be safe: `name="Alice"`

**Useful variable tricks:**

| Command | What It Does |
|---------|--------------|
| `name="Alice"` | Create a variable |
| `echo "$name"` | Print its value |
| `readonly name="Alice"` | Make it unchangeable |
| `unset name` | Delete the variable |
| `DATE=$(date)` | Store output of a command in a variable |
| `export NAME="Alice"` | Share variable with other programs |

**Special variables (always available in scripts):**

| Variable | What It Holds |
|----------|---------------|
| `$0` | The name of the script itself |
| `$1`, `$2` | First and second argument passed to the script |
| `$#` | Total number of arguments passed |
| `$?` | Exit status of the last command (0 = success) |
| `$$` | Process ID of current shell |
| `"$@"` | All arguments, each safely quoted |

Example — a script that greets by name:
```bash
#!/bin/bash
# Run as: ./greet.sh Alice

echo "Hello, $1!"
```

```bash
./greet.sh Alice
# Output: Hello, Alice!
```

### Step 4: Read Input from the User

Use `read` to ask the user to type something:

```bash
#!/bin/bash

read -p "What is your name? " name
echo "Hello, $name!"
```

Output:
```
What is your name? Bob
Hello, Bob!
```

| Command | What It Does |
|---------|--------------|
| `read name` | Wait for user to type, save to `name` |
| `read -p "Prompt: " name` | Show a prompt before waiting |
| `echo "text"` | Print text |
| `printf "Name: %s\n" "$name"` | Formatted print (more control) |

### Step 5: Make Decisions with If / Else

An **if statement** lets your script do different things based on a condition.

```bash
#!/bin/bash

age=18

if [ $age -ge 18 ]; then
    echo "You are an adult"
else
    echo "You are a minor"
fi
```

**Structure:**
```bash
if [ condition ]; then
    # runs if condition is TRUE
else
    # runs if condition is FALSE
fi
```

**Common conditions:**

| Condition | Meaning |
|-----------|---------|
| `[ "$a" = "$b" ]` | Are two strings equal? |
| `[ "$a" != "$b" ]` | Are two strings NOT equal? |
| `[ -z "$value" ]` | Is the string empty? |
| `[ -n "$value" ]` | Is the string NOT empty? |
| `[ $a -eq $b ]` | Are two numbers equal? |
| `[ $a -gt $b ]` | Is `a` greater than `b`? |
| `[ $a -lt $b ]` | Is `a` less than `b`? |
| `[ -f file.txt ]` | Does this file exist? |
| `[ -d folder/ ]` | Does this folder exist? |

**Multiple conditions:**
```bash
if [[ $age -ge 18 && $age -lt 60 ]]; then
    echo "Working age adult"
fi
```

**Case statement** — like a multi-way if:
```bash
day="Monday"

case "$day" in
  "Monday")   echo "Start of work week" ;;
  "Friday")   echo "End of work week"   ;;
  "Saturday") echo "Weekend!"           ;;
  *)          echo "Regular day"        ;;
esac
```

### Step 6: Repeat Tasks with Loops

Loops let you run the same code many times automatically.

#### For Loop — loop through a list

```bash
for fruit in apple banana mango; do
    echo "I like $fruit"
done
```

Output:
```
I like apple
I like banana
I like mango
```

#### For Loop — loop through numbers

```bash
for i in {1..5}; do
    echo "Number: $i"
done
```

#### While Loop — repeat while condition is true

```bash
count=1

while [ $count -le 3 ]; do
    echo "Count is: $count"
    ((count++))
done
```

Output:
```
Count is: 1
Count is: 2
Count is: 3
```

#### Loop through lines in a file

```bash
while read -r line; do
    echo "$line"
done < names.txt
```

**Loop control:**

| Command | What It Does |
|---------|--------------|
| `break` | Exit the loop immediately |
| `continue` | Skip this iteration, go to next |

### Step 7: Organize Code with Functions

A **function** is a block of code you name and reuse whenever you need it.

```bash
#!/bin/bash

# Define the function
greet() {
    echo "Hello, $1!"
}

# Call the function
greet "Alice"
greet "Bob"
```

Output:
```
Hello, Alice!
Hello, Bob!
```

**Function with return value:**
```bash
add() {
    local result=$(( $1 + $2 ))
    echo $result                    # return by printing
}

sum=$(add 5 3)
echo "Sum is: $sum"                 # Sum is: 8
```

| Concept | Explanation |
|---------|-------------|
| `my_func() { ... }` | Define a function |
| `my_func` | Call/run the function |
| `$1`, `$2` | Arguments passed to the function |
| `local x="val"` | Variable only exists inside this function |
| `return 1` | Exit function with a status code |

### Step 8: Work with Strings and Numbers

**String operations:**

```bash
name="hello world"

echo ${#name}           # length: 11
echo ${name^^}          # uppercase: HELLO WORLD
echo ${name,,}          # lowercase: hello world
```

**Arithmetic:**

```bash
a=10
b=3

echo $((a + b))         # 13
echo $((a - b))         # 7
echo $((a * b))         # 30
echo $((a / b))         # 3
echo $((a % b))         # 1  (remainder)

((a++))                 # increment a by 1
((a--))                 # decrement a by 1
```

### Step 9: Arrays (Lists)

An **array** holds multiple values under one variable name.

```bash
fruits=("apple" "banana" "mango")

echo "${fruits[0]}"     # apple (first item, index starts at 0)
echo "${fruits[1]}"     # banana
echo "${fruits[@]}"     # all items
echo "${#fruits[@]}"    # total count: 3
```

Loop through an array:
```bash
for fruit in "${fruits[@]}"; do
    echo "$fruit"
done
```

### Step 10: Redirect Output and Use Pipes

**Redirection** — send output to a file instead of the terminal:

```bash
echo "Hello" > output.txt       # write to file (overwrites)
echo "World" >> output.txt      # append to file
cat < output.txt                # read from file
ls 2> errors.log                # save only errors to file
ls > all.log 2>&1               # save both output and errors
```

**Pipes** — send output of one command as input to another:

```bash
ls | grep ".txt"                # list files, filter only .txt ones
cat names.txt | sort            # print file, then sort lines
echo "hello world" | wc -w     # count words: 2
cmd | tee output.txt            # show output AND save to file
```

Think of `|` as "pass this to the next command".

---

### Step 11: Exit Codes and Error Handling

Every command returns an **exit code** when it finishes:
- `0` means **success**
- Any other number means **failure**

```bash
ls /tmp
echo $?            # prints 0 (success)

ls /fakepath
echo $?            # prints non-zero (error)
```

**Stop script on first error (recommended for safety):**
```bash
#!/bin/bash
set -euo pipefail
# -e  → exit on error
# -u  → error on undefined variable
# -o pipefail → catch errors in pipes
```

**Exit the script with a code:**
```bash
exit 0             # script finished successfully
exit 1             # script finished with an error
```

---

### Step 12: Debug Your Script

When something goes wrong, use these to find the problem:

```bash
bash -n script.sh     # check syntax only (don't run)
bash -x script.sh     # run and print each step before executing
```

Or add inside the script:
```bash
set -x        # start showing each step
# ... your code ...
set +x        # stop showing steps
```

Example output of `bash -x`:
```
+ name=Alice
+ echo 'Hello, Alice'
Hello, Alice
```

The `+` means Bash is showing you what it ran before running it.

---

### Step 13: Reload Shell Config

When you edit `~/.bashrc` (your shell config file), you need to reload it:

```bash
source ~/.bashrc      # reload Bash config
. ~/.bashrc           # same thing, shorter form
source ~/.profile     # reload profile settings
```

## Safety Flags — Always Use These

Add this at the top of every script you write:

```bash
#!/bin/bash
set -euo pipefail
```

| Flag | What It Does |
|------|--------------|
| `-e` | Stop the script if any command fails |
| `-u` | Stop if you use a variable that was never set |
| `-o pipefail` | Stop if any command in a pipe fails |

Without these, a script can silently fail and keep running — causing unexpected damage.

## Quick Reference — All Key Syntax

```bash
#!/bin/bash
set -euo pipefail

# Variables
name="Alice"
echo "$name"
DATE=$(date)

# User input
read -p "Enter name: " name

# If / else
if [ "$name" = "Alice" ]; then
    echo "Welcome Alice"
elif [ -z "$name" ]; then
    echo "Name is empty"
else
    echo "Hello $name"
fi

# For loop
for i in {1..5}; do
    echo $i
done

# While loop
while [ $count -le 10 ]; do
    ((count++))
done

# Function
greet() {
    echo "Hello, $1"
}
greet "Bob"

# Redirect & pipe
echo "text" > file.txt
cat file.txt | grep "text"

# Arithmetic
sum=$((5 + 3))

# Array
items=("a" "b" "c")
echo "${items[@]}"

# Debug
bash -x script.sh

# Exit
exit 0
```
