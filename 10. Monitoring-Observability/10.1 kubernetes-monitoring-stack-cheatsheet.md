# Kubernetes Monitoring Stack Cheatsheet 

> If you are new to Kubernetes monitoring, read this from top to bottom once. It explains the complete idea in simple language.

---

## Goal

You wanted monitoring in Kubernetes, so you installed:

```text
Prometheus
Grafana
Loki
Promtail
Alertmanager
```

Why install all these tools?

Because you want answers to questions like:

```text
Is my application healthy?
How much CPU is used?
How much memory is used?
What errors are coming?
When did a pod crash?
```

---

## Part 1: What Happened When You Ran Helm Install?

You ran:

```bash
helm install prometheus prometheus-community/kube-prometheus-stack -n monitoring
```

### Step 1: You asked Helm to install

```text
You
 |
 v
Helm
```

Think of Helm as an installer for Kubernetes apps.

### Step 2: Helm went to the repository

```text
Helm
 |
 v
Prometheus Repository
```

Think of this like an app store for Kubernetes packages.

### Step 3: Helm downloaded a chart

A chart is a ready-made Kubernetes package.

```text
Chart
 |
 +-- deployment.yaml
 +-- service.yaml
 +-- secret.yaml
 +-- configmap.yaml
 +-- many more
```

### Step 4: Helm sent YAML files to Kubernetes API

```text
Helm
 |
 v
Kubernetes API
```

### Step 5: Kubernetes created Deployments and Pods

```text
Deployment
 |
 v
Pod
```

Examples:

```text
Prometheus Pod
Grafana Pod
Loki Pod
```

### Step 6: Pods needed container images

```text
Pod
 |
 v
Need Image
```

### Step 7: Container runtime downloaded images

Kubernetes nodes downloaded images from registries such as `quay.io` and `docker.io`.

```text
Internet
 |
 v
quay.io / docker.io
 |
 v
Image Download
```

### Step 8: Image became container inside pod

```text
Image
 |
 v
Container
 |
 v
Pod
```

### Step 9: Pod started running

```text
Container
 |
 v
Running Pod
```

### Complete Installation Flow

```text
You
 |
 v
Helm Install
 |
 v
Chart Download
 |
 v
YAML Files
 |
 v
Kubernetes
 |
 v
Deployments
 |
 v
Pods
 |
```

## Part 3: What Does Each Component Do?

### 1) Prometheus

Think: Data Collector for metrics.

It collects values like:

```text
CPU
Memory
Disk
Network
```

Flow:

```text
Node / Kubernetes metrics
 |
 v
Prometheus
```

Example data:

```text
CPU = 20%
Memory = 40%
```

Prometheus stores this as time-series data.

### 2) Grafana

Think: Dashboard.

Prometheus has data. Grafana turns that data into graphs.

```text
Prometheus
 |
 v
Grafana
 |
 v
Graphs and Dashboards
```

Without Grafana: plain numbers.
With Grafana: clear visual dashboards.

### 3) Loki

Think: Log storage.

Stores logs such as:

```text
Database Error
API Error
Application Crash
```

```text
Logs
 |
 v
Loki
```

### 4) Promtail

Think: Log courier.

Promtail reads logs from pods/nodes and sends them to Loki.

```text
Pod Logs
 |
 v
Promtail
 |
 v
Loki
```

### 5) Alertmanager

Think: Security guard for alerts.

If something bad happens (CPU 100%, pod crash), Alertmanager sends notifications.

```text
Prometheus Alert
 |
 v
Alertmanager
 |
 v
Email / Slack / Webhook
```

### 6) Node Exporter

Think: Node health collector.

Collects node-level metrics:

```text
CPU
Memory
Disk
Network
```

```text
Node
 |
 v
Node Exporter
 |
 v
Prometheus
```

### 7) kube-state-metrics

Think: Kubernetes object status collector.

It answers:

```text
How many pods?
How many deployments?
How many nodes?
```

```text
Kubernetes API Objects
 |
 v
kube-state-metrics
 |
 v
Prometheus
```

## Complete Monitoring Architecture (Easy to Remember)

```text
                 Grafana
                    ^
                    |
        +-----------+-----------+
        |                       |
   Prometheus               Loki
        ^                     ^
        |                     |
Node Exporter           Promtail
        ^                     ^
        |                     |
      Nodes               Pod Logs
```

Extra flow:

```text
Prometheus
 |
 v
Alertmanager
 |
 v
Email / Slack
```

## Access Grafana (Quick)

Get password:

```bash
kubectl get secret -n monitoring prometheus-grafana \
  -o jsonpath="{.data.admin-password}" | base64 --decode && echo
```

Port-forward:

```bash
kubectl port-forward -n monitoring svc/prometheus-grafana 3000:80
```

Open:

```text
http://localhost:3000
username: admin
password: output of command above
```
