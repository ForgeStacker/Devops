# Domain Registration & DNS Setup — Cheatsheet

---

## Introduction

When you buy a domain like `yourname.com`, it is just a name — it does not point anywhere yet.

To make that domain load your website, you need to connect the domain to a server. This is done through **DNS** (Domain Name System).

**DNS** is like a phone book for the internet. When someone types `yourname.com` in their browser, DNS looks up the IP address of the server where your website lives and sends the request there.

This cheatsheet explains the full journey:

```
Buy Domain → Set Nameservers → Configure DNS Records → Visitors reach your site
```

**Key players:**

| Term | Role |
|------|------|
| **Registrar** | Where you buy the domain (Hostinger, GoDaddy, Namecheap) |
| **Nameserver** | Tells the internet who manages DNS for your domain |
| **Cloudflare** | Acts as your DNS manager — stores and serves your DNS records |
| **DNS Record** | A rule that maps a domain name to a server IP or another address |
| **Web Server** | Where your actual website files live (Hostinger, Vercel, any server) |

---

## Aims — What We Want to Achieve

1. **Buy a domain** from a registrar
2. **Hand over DNS control to Cloudflare** by setting Cloudflare nameservers
3. **Create DNS records** in Cloudflare that point your domain to your server
4. **Understand how a visitor's request flows** from browser to your website

---

## How to Achieve It — Step by Step

---

### Step 1: Buy a Domain from a Registrar

A **registrar** is a company that sells domain names.

Popular registrars:
- **Hostinger** — hostinger.com
- **GoDaddy** — godaddy.com
- **Namecheap** — namecheap.com

When you buy a domain, the registrar stores:
- Your domain name (e.g. `yourname.com`)
- Expiration date (when you need to renew)
- Nameservers (who manages DNS for this domain)

By default the registrar assigns their own nameservers (e.g. `ns1.dns-parking.com`). We will replace these with Cloudflare's nameservers in the next step.

---

### Step 2: Set Cloudflare Nameservers in Your Registrar

**Nameservers** tell the internet: *"Go ask THIS server about DNS records for this domain."*

We want Cloudflare to manage our DNS — so we point nameservers to Cloudflare.

**Add your domain to Cloudflare first:**
1. Go to [dash.cloudflare.com](https://dash.cloudflare.com)
2. Click **Add a Site** → enter your domain (e.g. `yourname.com`)
3. Choose the Free plan
4. Cloudflare will give you two nameservers, e.g.:
   ```
   dora.ns.cloudflare.com
   jack.ns.cloudflare.com
   ```
   (yours will be different — Cloudflare assigns unique ones per account)

**Now set these in your registrar (e.g. Hostinger):**

```
Hostinger Dashboard
  → Domains
  → Select your domain
  → Nameservers
  → Change to Custom Nameservers
  → Enter:
       dora.ns.cloudflare.com
       jack.ns.cloudflare.com
  → Save
```

**What happens next:**
- Cloudflare becomes the DNS manager for your domain
- All DNS lookups for `yourname.com` now go through Cloudflare
- DNS propagation takes a few minutes to 48 hours globally

---

### Step 3: Create DNS Records in Cloudflare

DNS records are the rules that tell Cloudflare where to send traffic.

Go to **Cloudflare Dashboard → your domain → DNS → Records**

#### Common DNS Record Types

| Type | Purpose | Example |
|------|---------|---------|
| **A** | Points domain to a server IPv4 address | `yourname.com` → `76.76.21.21` |
| **CNAME** | Points domain to another domain name | `www` → `cname.vercel-dns.com` |
| **MX** | Points to mail server (for email) | `yourname.com` → mail server |
| **TXT** | Stores text — used for verification, SPF, DKIM | Google/SSL verification |

#### Example DNS Records

| Type | Name | Content | Proxy | TTL |
|------|------|---------|-------|-----|
| A | `@` | `76.76.21.21` | Proxied | Auto |
| CNAME | `www` | `cname.vercel-dns.com` | Proxied | Auto |

- `@` means the root domain itself (`yourname.com`)
- `www` means `www.yourname.com`
- **Proxied** means traffic goes through Cloudflare (enables DDoS protection, caching, HTTPS)
- **DNS only** means traffic goes directly to the server (Cloudflare not in the path)

#### Common Setups

**Pointing to a VPS / server IP:**
```
A    @      <your-server-ip>    Proxied    Auto
A    www    <your-server-ip>    Proxied    Auto
```

**Pointing to Vercel:**
```
A      @      76.76.21.21           Proxied    Auto
CNAME  www    cname.vercel-dns.com  Proxied    Auto
```

**Pointing to a subdomain (e.g. app.yourname.com → server):**
```
A    app    <your-server-ip>    Proxied    Auto
```

### Step 4: How a Visitor's Request Flows

Once DNS is set up, here is what happens every time someone visits your site:

```
1. Visitor types yourname.com in browser

2. Browser asks the internet: "What is the IP of yourname.com?"

3. Internet checks nameservers for yourname.com
   → Nameservers say: "Ask Cloudflare"

4. Cloudflare checks its DNS records for yourname.com
   → Finds: A record → 76.76.21.21 (your server IP)
   → Returns this IP to the browser

5. Browser sends the HTTP request to 76.76.21.21
   (your web server: Hostinger / Vercel / any server)

6. Web server receives the request
   → Sends back your website files (HTML, CSS, JS)

7. Browser renders the page
   → Visitor sees your website
```

## Full Scenario — Putting It All Together

Here is the complete real-world flow for setting up `yourname.com`:

| Step | Action | Where |
|------|--------|--------|
| 1 | Buy `yourname.com` | Hostinger / GoDaddy / Namecheap |
| 2 | Add site to Cloudflare | dash.cloudflare.com |
| 3 | Copy Cloudflare nameservers | Cloudflare dashboard |
| 4 | Set nameservers in registrar | Hostinger → Domains → Nameservers |
| 5 | Wait for propagation | A few minutes to 48 hours |
| 6 | Create DNS A/CNAME records | Cloudflare → DNS → Records |
| 7 | Visitors request domain | Cloudflare reads DNS → sends to server |
| 8 | Website loads for visitor | Web server responds |

---

## Key Points to Remember

- **Nameservers decide who manages DNS.** By setting Cloudflare nameservers, Cloudflare becomes the DNS authority for your domain.
- **Cloudflare stores DNS Records** (A, CNAME, MX, TXT, etc.) — not the registrar anymore.
- **DNS Records decide where subdomains point.** Each subdomain can point to a different server.
- **DNS propagation takes time** — changes can take up to 48 hours to spread globally (usually much faster).
- **Proxied vs DNS Only** — Proxied routes traffic through Cloudflare (adds protection + HTTPS). DNS Only goes straight to your server.
