# Vercel Frontend Deployment — Cheatsheet

---

## Introduction

**Vercel** is a cloud platform for deploying frontend web applications. It is built specifically for frameworks like React, Vite, Next.js, and others.

You write code on your laptop → push it to GitHub → Vercel automatically builds and deploys it to a global CDN. No server setup, no manual deploy commands.

Every time you push new code, Vercel detects the change (via a GitHub webhook), rebuilds your app, and publishes the new version — automatically, in seconds.

Your site is live at a Vercel URL like `https://my-app.vercel.app` by default. You can then connect your own custom domain.

**Technologies in this stack:**

| Technology | Role |
|------------|------|
| Vite + TypeScript | Frontend build tool and language |
| Git | Local version control |
| GitHub | Cloud code storage, triggers deployments |
| Vercel | Builds and hosts the frontend on a global CDN |
| Hostinger (DNS) | Manages DNS records for custom domain |
| Let's Encrypt | Issues free SSL certificate (via Vercel) |

---

## Aims — What We Want to Achieve

1. **Build a frontend app locally** with Vite + TypeScript
2. **Push code to GitHub** — store it safely with version history
3. **Deploy automatically via Vercel** — every push goes live
4. **Connect a custom domain** — replace `*.vercel.app` with your own domain
5. **Enable HTTPS** — secure the site with SSL automatically
6. **Set up CI/CD** — every future push auto-builds and deploys

---

## How to Achieve It — Step by Step

---

### Step 1: Develop on Your Laptop

Create a Vite + TypeScript project and test it locally.

```bash
npm create vite@latest my-app    # create new Vite project
# Select: React + TypeScript (or Vanilla + TypeScript)
cd my-app
npm install                      # install dependencies
npm run dev                      # start local dev server
```

Your app runs at `http://localhost:5173` — only on your machine.

**Project folder structure:**
```
my-app/
├── src/              # your source code
├── public/           # static assets
├── index.html
├── package.json
├── vite.config.ts
└── tsconfig.json
```

**Development cycle:**
```
Make code change → Save → Test in browser → git commit → git push
```

Stop the dev server when done:
```bash
Ctrl + C
```

---

### Step 2: Push Code to GitHub

GitHub stores your code and acts as the trigger for Vercel deployments.

```bash
git init                                      # initialize git
git add .                                     # stage all files
git commit -m "Initial commit"                # save snapshot

# Go to github.com → create a new repository
# Copy the repo URL, then:

git remote add origin <github-repo-url>       # link to GitHub repo
git push -u origin main                       # push code to GitHub
```

Your code is now on GitHub. Vercel will watch this repo.

**What GitHub provides:**
- Stores your code safely in the cloud
- Version control — full history of every change
- Collaboration — others can contribute
- Triggers Vercel deployment on every push (via Webhook)

---

### Step 3: Connect Vercel to GitHub (One-Time Setup)

**Login and import your repo:**

1. Go to [vercel.com](https://vercel.com) → **Login with GitHub**
2. Click **Add New Project** → **Import Git Repository**
3. Select your GitHub repo
4. Select the **root directory** (the folder where `package.json` lives)
   - e.g. if your project is inside a subfolder, select that subfolder
5. Vercel auto-detects Vite — confirms build settings:

| Setting | Value |
|---------|-------|
| Framework | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |

6. Click **Deploy**

**Vercel build process (automatic):**
```
Clone repo from GitHub
  → npm install          (install dependencies)
  → npm run build        (build project)
  → Output: dist/        (production-ready static files)
  → Deploy to Global CDN
```

Your site is live at:
```
https://<project-name>.vercel.app
```

---

### Step 4: How Future Updates Work (CI/CD)

After the one-time setup, every code change follows this automatic cycle:

```
You edit code on laptop
  ↓
git add .
git commit -m "update"
git push
  ↓
GitHub receives new changes
  ↓
GitHub sends Webhook notification to Vercel
  ↓
Vercel detects the new push
  ↓
Auto build starts: npm install + npm run build
  ↓
New version deploys to Global CDN
  ↓
Live on your domain — automatically, in seconds
```

No manual deploy step. Just push to GitHub.

### Step 5: Connect a Custom Domain

By default your site is at `*.vercel.app`. To use your own domain (e.g. `yourdomain.com`):

**In Vercel — get the DNS records:**

Go to your project → **Settings** → **Domains** → **Add Domain**

Vercel provides two DNS records to copy:

| Type | Name | Value |
|------|------|-------|
| A | `@` | `76.76.21.21` |
| CNAME | `www` | `cname.vercel-dns.com` |

**In Hostinger (or your registrar) — add these records:**

1. Go to **Hostinger Dashboard → Domains → your domain → DNS / Nameservers → Manage DNS**
2. Add the A record:
   - Type: `A` | Name: `@` | Value: `76.76.21.21` | TTL: `14400`
3. Add the CNAME record:
   - Type: `CNAME` | Name: `www` | Value: `cname.vercel-dns.com` | TTL: `14400`
4. Save Records

**DNS Propagation:**
- Takes a few minutes to a few hours
- Internet-wide DNS servers update to the new records

Your domain now points to your Vercel deployment.

---

### Step 6: HTTPS / SSL (Automatic)

Once the domain is verified:

1. **Vercel verifies domain ownership** (checks DNS records)
2. **Let's Encrypt issues an SSL certificate** automatically
3. **Vercel enables HTTPS** for your domain

Result:
- Your site is live at `https://yourdomain.com`
- Connection is encrypted
- Browser shows the padlock — trusted and secure
- No manual certificate setup needed — ever

---

### Step 7: Users Access Your Site

Once live, people anywhere in the world can access your site:

```
User opens https://yourdomain.com
  → DNS resolves domain to Vercel's CDN
  → Nearest Vercel edge server responds
  → HTML, CSS, JS files served instantly
  → User sees your website
```

Vercel's global CDN ensures the site loads fast from anywhere.
